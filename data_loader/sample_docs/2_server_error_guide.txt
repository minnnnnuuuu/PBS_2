[문서 제목: 서버 공통 에러 해결 가이드 v1.0]

이 문서는 서버 운영 중 발생하는 주요 HTTP 에러 코드에 대한 대응 방법을 기술합니다.

1. 500 Internal Server Error (내부 서버 오류)
   - 원인: 백엔드 코드의 문법 오류, DB 연결 실패, 혹은 권한 문제.
   - 해결 방법:
     (1) 서버 로그(/var/log/syslog)를 확인하여 정확한 에러 메시지를 파악한다.
     (2) 데이터베이스 서비스가 정상 작동 중인지 확인한다 (systemctl status mysql).
     (3) 최근 배포된 코드에 오타가 없는지 검토하고 롤백을 고려한다.

2. 502 Bad Gateway (잘못된 게이트웨이)
   - 원인: 앞단의 웹 서버(Nginx)가 뒷단의 WAS(Tomcat/Gunicorn)로부터 응답을 받지 못함.
   - 해결 방법:
     (1) 백엔드 애플리케이션 서버가 죽어있는지 확인하고 재시작한다.
     (2) Nginx 설정 파일에서 proxy_pass 주소가 올바른지 확인한다.
     (3) 방화벽(Security Group)이 포트를 막고 있는지 점검한다.

3. 504 Gateway Timeout (시간 초과)
   - 원인: 백엔드 처리 시간이 너무 오래 걸려서 연결이 끊김.
   - 해결 방법: 대용량 처리 로직을 최적화하거나, Nginx의 timeout 설정을 늘린다.
